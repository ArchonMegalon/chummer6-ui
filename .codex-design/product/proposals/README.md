# Design petitions

This directory is the legal upward path when a worker, reviewer, or auditor cannot proceed without changing canonical design truth.

Use a petition when:

* a repo boundary blocks valid work
* a shared contract seam is missing
* mirrored canon contradicts reality
* a worker would otherwise be forced to invent local truth

Petitions are not implementation patches.
They are proposal packets for `chummer6-design`.

Required fields:

* blocked repo
* violated boundary or missing seam
* why current canon is insufficient
* workaround attempts rejected
* proposed resolution
* affected canonical files
* urgency

Repeated findings with the same root cause should be synthesized into one clearer petition rather than published as parallel duplicates.
