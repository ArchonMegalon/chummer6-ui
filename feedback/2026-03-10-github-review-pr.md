# GitHub Codex Review

PR: https://github.com/ArchonMegalon/chummer-presentation/pull/1

Findings:
- [medium] Chummer.Blazor/Components/Shared/GeneratedAssetReviewPanel.razor : line 40 Each asset tab now advertises `aria-controls="generated-asset-panel-<id>"`, but the component only renders a single `role="tabpanel"` for the currently selected asset at lines 48-53. Every non-selected tab therefore points to a panel that does not exist in the DOM, which is invalid tab semantics and will confuse assistive tech. Render hidden panels for inactive assets or stop emitting broken `aria-controls` targets, and add a regression test that verifies all tabs reference an existing panel.
- [medium] scripts/ai/milestones/b13-accessibility-signoff-check.sh : line 120 The B13 signoff gate switches to `dotnet test --no-restore --no-build` solely because `Chummer.Tests/bin` and `Chummer.Tests/obj` exist. That allows stale test binaries from an earlier checkout/build to satisfy the signoff without exercising the current source changes, which is an offline-state hazard for the claimed runnable closure. The gate needs a freshness check or a build-backed path for signoff runs instead of assuming any existing output tree is current.
- [medium] scripts/ai/milestones/b7-browser-isolation-check.sh : line 205 The B7 script still exits successfully when the portal target is unreachable in non-CI environments, printing only a note that the runtime deployment probe was skipped. That means local `verify.sh` can report final browser/deployment signoff complete without ever checking deep-link refresh, COOP/COEP headers, wasm MIME, or service-worker behavior on a live deployment. For a final deployment signoff slice, this should fail unless the caller explicitly opts into a documented skip mode.
